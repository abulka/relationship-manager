No, sets and maps do not allow duplicate keys.  Because of that possibility,
you are stuck with lists and arrays.  To use Maps, you would have to alter
each key value somehow by appending a count or something to keep them
unique.  Otherwise, you could store a simple pair class you create in an
ArrayList but you would have to consider number of records and performance
issues which depends on your requirements.

How about using a Hashtable (Or HashMap) with its key part being your
key itself and the value component being an ArrayList containing your
values ? 

While adding key-value pairs, you need to first check whether the key
already exists in the Hashtable by using get(key) method of the
Hashtable. If the value returned is non-null, cast the returned result
into an ArrayList and add your new value to it. If it is null, create a
new ArrayList, add the value to it and store it in the hashtable.

Similar logic to be applied for deletion of key-value pairs also (you
delete the entry only if the returned ArrayList has a size of exactly 1.
Otherwise you just delete the value entry from the ArrayList - not from
the hashtable)
